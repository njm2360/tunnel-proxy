# encrypt-proxy アーキテクチャドキュメント

## 概要

encrypt-proxy は Go 製の暗号化トンネルプロキシ。
ブラウザ/アプリのトラフィックを PSK（事前共有鍵）ベースの AEAD 暗号化トンネルで保護しながらインターネットへ中継する。

---

## 全体フロー

```
ブラウザ/アプリ
    │  HTTP/HTTPS プロキシ設定 (127.0.0.1:8080)
    ▼
┌─────────────────────────────┐
│  クライアントプロセス         │
│                             │
│  ローカル HTTP プロキシ      │  ← CONNECT/HTTP/WebSocket/SSE に対応
│  HTTPS MITM (CA 証明書)     │  ← TLS を終端して復号
│                             │
│  smux 多重化                │  ← リクエストごとに独立した仮想ストリーム
│  AES-256-GCM 暗号化         │  ← フレーム単位で暗号化
│  WebSocket トランスポート    │
└────────────┬────────────────┘
             │  wss:// または ws://
             │  (上流プロキシがある場合は CONNECT でトンネル)
             ▼
┌─────────────────────────────┐
│  サーバープロセス            │
│                             │
│  WebSocket 受け付け         │
│  AES-256-GCM 復号           │
│  smux ストリーム受理         │
│  HTTP/HTTPS/WS/WSS リレー   │  ← 宛先へ実際のリクエストを転送
└────────────┬────────────────┘
             │  平文 HTTP/HTTPS/WebSocket
             ▼
      インターネット上の宛先サーバー
```

## 接続フロー詳細

### トランスポート層：クライアント ↔ サーバー間

クライアントとサーバーの間は **1本の WebSocket 接続** を常時維持し、その上に smux で仮想ストリームを多重化する。ブラウザのリクエストごとにストリームを1本開き、レスポンスが終わったら閉じる。

```
物理接続（WebSocket / TCP 1本）
  └── AES-256-GCM 暗号化フレーム
        └── smux 多重化セッション
              ├── Stream #1  ─ HTTP リクエスト A（GET example.com）
              ├── Stream #2  ─ HTTPS リクエスト B（POST api.example.com）
              └── Stream #3  ─ WebSocket 接続 C（wss://ws.example.com）
```

各ストリームは独立したカウンターを持つため、並列リクエストがあっても互いに干渉しない。

### ケース別の接続フロー

#### 1. 平文 HTTP（`http://example.com`）

```
ブラウザ                    クライアント                      サーバー           宛先
  │                            │                               │                │
  │ GET http://example.com/    │                               │                │
  │ HTTP/1.1 (絶対URL形式)     │                               │                │
  │───────────────────────────→│                               │                │
  │                            │ smux.OpenStream()             │                │
  │                            │ StreamHeader{HTTP, example.com:80}             │
  │                            │ GET /  HTTP/1.1               │                │
  │                            │──────────────────────────────→│                │
  │                            │                               │ TCP Dial       │
  │                            │                               │───────────────→│
  │                            │                               │ GET / HTTP/1.1 │
  │                            │                               │───────────────→│
  │                            │         200 OK + Body         │                │
  │                            │←──────────────────────────────│                │
  │ 200 OK + Body              │                               │                │
  │←───────────────────────────│                               │                │
```

ブラウザは Keep-Alive でこの接続を使い回すため、後続リクエストも同じ TCP 接続でループ処理される。

---

#### 2. HTTPS（`https://example.com`）— MITM あり

```
ブラウザ                    クライアント                      サーバー           宛先
  │                            │                               │                │
  │ CONNECT example.com:443    │                               │                │
  │───────────────────────────→│                               │                │
  │ 200 OK                     │                               │                │
  │←───────────────────────────│                               │                │
  │                            │                               │                │
  │ TLS ClientHello (0x16)     │ Peek(1) で検知                │                │
  │───────────────────────────→│ CA でリーフ証明書を動的生成   │                │
  │ TLS Handshake (MITM)       │                               │                │
  │←──────────────────────────→│                               │                │
  │                            │                               │                │
  │ GET / HTTP/1.1 (復号済み)  │ smux.OpenStream()             │                │
  │───────────────────────────→│ StreamHeader{HTTPS, example.com:443}           │
  │                            │──────────────────────────────→│                │
  │                            │                               │ TLS Dial       │
  │                            │                               │  (SNI付き)     │
  │                            │                               │───────────────→│
  │                            │                               │ GET / HTTP/1.1 │
  │                            │                               │───────────────→│
  │                            │         200 OK + Body         │                │
  │                            │←──────────────────────────────│                │
  │ 200 OK + Body (再暗号化)   │                               │                │
  │←───────────────────────────│                               │                │
```

ブラウザ↔クライアント間は MITM TLS、クライアント↔サーバー間は smux+AES、サーバー↔宛先間は正規の TLS という3層構造になる。

#### 3. WebSocket（wss://ws.example.com）

```
ブラウザ                    クライアント                      サーバー           宛先
  │                            │                               │                │
  │ CONNECT ws.example.com:443 │                               │                │
  │───────────────────────────→│                               │                │
  │ 200 OK                     │                               │                │
  │←───────────────────────────│                               │                │
  │ TLS Handshake (MITM)       │                               │                │
  │←──────────────────────────→│                               │                │
  │                            │                               │                │
  │ GET / HTTP/1.1             │ smux.OpenStream()             │                │
  │ Upgrade: websocket         │ StreamHeader{WSS, ws.example.com:443}          │
  │───────────────────────────→│──────────────────────────────→│                │
  │                            │                               │ TLS+WS Dial    │
  │                            │                               │───────────────→│
  │                            │                               │ GET / Upgrade  │
  │                            │                               │───────────────→│
  │                            │   101 Switching Protocols     │                │
  │                            │←──────────────────────────────│                │
  │ 101 Switching Protocols    │                               │                │
  │←───────────────────────────│                               │                │
  │                            │                               │                │
  │  WS フレーム（双方向）      │   生バイトストリーム bridge    │                │
  │←──────────────────────────→│←─────────────────────────────→│←──────────────→│
```

101 以降は HTTP フレームを持たない生バイトストリームになるため、両端を goroutine でそのままコピーする（BridgeWebSocket）。

#### 4. 各データが何に包まれて流れるか

```
ブラウザ → クライアント（ローカル接続）
  平文 HTTP リクエスト / TLS（MITM終端）

クライアント → サーバー（インターネット）
  WebSocket フレーム
    └── AES-256-GCM 暗号化フレーム [4B長さ][暗号文+タグ]
          └── smux フレーム
                └── StreamHeader [種別][ホスト長][ホスト:ポート]
                      └── 平文 HTTP リクエスト / WebSocket フレーム

サーバー → 宛先（インターネット）
  平文 TCP または TLS（正規証明書）
```

## 暗号化・ハンドシェイク

### ハンドシェイクシーケンス

```
クライアント                                サーバー
    │                                          │
    │── [32B X25519公開鍵][4B clientSalt] ──→  │  (平文)
    │← [32B X25519公開鍵][4B serverSalt] ───  │  (平文)
    │                                          │
    │  両者が独立に計算:                        │
    │  dhShared  = X25519(myPriv, theirPub)    │
    │  sessionID = SHA-256(                    │
    │    clientEphPub || serverEphPub          │
    │    || clientSalt || serverSalt)          │
    │  sessionKey = HKDF-SHA256(               │
    │    IKM  = dhShared,                     │
    │    salt = sessionID,                    │
    │    info = opaque[16B])                  │
    │                                          │
    │← [暗号化: 64B サーバーEd25519署名] ────  │  サーバー身元証明
    │  クライアント: serverPubKey で署名検証    │
    │                                          │
    │── [暗号化: 32B クライアントEd25519公開鍵  │
    │          + 64B クライアントEd25519署名] →│  クライアント身元証明
    │                  サーバー: 署名検証 +    │
    │                  authorized_keys 確認   │
    │                                          │
    │    以降: smux 多重化 + AEAD 暗号化        │
```

### セキュリティ特性

| 特性             | 実現方法                                                                       |
| ---------------- | ------------------------------------------------------------------------------ |
| サーバー認証     | クライアントが固定サーバー公開鍵で Ed25519 署名を検証                          |
| クライアント認証 | サーバーが authorized_keys で Ed25519 署名を検証。クライアント個別に失効可能   |
| 鍵交換           | X25519 ECDH（接続ごとにエフェメラル鍵を生成）                                  |
| Forward Secrecy  | DH共有シークレットをセッション鍵に混入。秘密鍵漏洩でも過去セッションは復号不可 |
| 暗号化           | AES-256-GCM（認証付き暗号、改ざん検知あり）                                    |
| ノンス管理       | `[4B 自salt固定][8B 単調増加カウンター]` でノンス再利用・リプレイ攻撃を防止    |
| リプレイ攻撃防止 | 署名対象が sessionID（セッション固有ハッシュ）のためシグネチャを再利用不可     |
| バイナリ解析耐性 | HKDF info を可読文字列でなく opaque バイト値で管理                             |

### HKDF info の役割

```
infoSessionKey (16B opaque) → dhShared + sessionID からセッション鍵を導出
```

sessionID は両エフェメラル公開鍵と両 salt の SHA-256 ハッシュで、
ハンドシェイク固有の値として鍵とノンス空間を分離する。

## フレーミング

暗号化 TCP 接続上でのフレームフォーマット:

```
┌──────────────────┬──────────────────────────────────────────┐
│  4B 暗号文長     │  暗号文 (平文 + 16B GCM 認証タグ)        │
└──────────────────┴──────────────────────────────────────────┘
```

- 長さフィールドは認証タグを含む暗号文バイト数
- 最大 4 MiB（メモリ枯渇攻撃防止のサニティキャップ）

---

## 多重化

1本の暗号化 TCP 接続を **smux**（xtaci/smux）で多重化する。

```
暗号化 TCP 接続
  └── smux セッション
        ├── ストリーム #1  (HTTP リクエスト A)
        ├── ストリーム #2  (HTTP リクエスト B)
        └── ストリーム #3  (WebSocket 接続 C)
```

各ストリームの先頭には宛先情報を示す **StreamHeader** を付与する:

```
┌──────────┬────────────┬──────────────────────────┐
│ 1B 種別  │ 2B ホスト長 │ ホスト:ポート文字列       │
└──────────┴────────────┴──────────────────────────┘

種別コード:
  0x01 = HTTP
  0x02 = HTTPS
  0x03 = WebSocket (ws://)
  0x04 = WebSocket Secure (wss://)
```

## HTTPS MITM の仕組み

```
ブラウザ                クライアントプロキシ              サーバー → 宛先
    │                         │                              │
    │ CONNECT example.com:443 │                              │
    │────────────────────────→│                              │
    │ 200 OK                  │                              │
    │←────────────────────────│                              │
    │                         │                              │
    │ TLS ClientHello         │                              │
    │────────────────────────→│ (0x16 を Peek で検知)        │
    │                         │ CA でリーフ証明書を動的生成  │
    │ TLS Handshake (MITM)    │                              │
    │←───────────────────────→│                              │
    │                         │                              │
    │ GET /path HTTP/1.1      │ smux ストリーム開通           │
    │────────────────────────→│──────────────────────────────→│
    │                         │                              │
```

- ローカル CA（`ca.crt`）をブラウザ/OS の信頼ストアに登録することで HTTPS サイトへのアクセスが可能になる
- `Peek(1)` で TLS ClientHello の先頭バイト `0x16` を検出し、平文接続と分岐

## 設定ファイル

### クライアント (`config/client.yaml`)

| キー                       | 説明                                                    | デフォルト       |
| -------------------------- | ------------------------------------------------------- | ---------------- |
| `listen_addr`              | ローカルプロキシの待ち受けアドレス                      | `127.0.0.1:8080` |
| `server_addr`              | サーバーのアドレス（ホスト:ポートまたは CF Tunnel URL） | —                |
| `ca_cert_path`             | MITM 用 CA 証明書のパス                                 | `./ca.crt`       |
| `ca_key_path`              | MITM 用 CA 秘密鍵のパス                                 | `./ca.key`       |
| `reconnect_delay_ms`       | 再接続初期遅延（ms）                                    | `1000`           |
| `max_reconnect_delay_ms`   | 再接続最大遅延（ms）                                    | `30000`          |
| `upstream_proxy.http_url`  | ws:// トンネル時に使う上流プロキシ URL                  | —                |
| `upstream_proxy.https_url` | wss:// トンネル時に使う上流プロキシ URL                 | —                |
| `upstream_proxy.ca_cert`   | プロキシへの TLS 接続に使うカスタム CA 証明書パス       | —                |
| `upstream_proxy.insecure`  | プロキシへの TLS 証明書検証をスキップ（開発用）         | `false`          |
| `tls_ca_cert_path`         | サーバー TLS の検証に使うカスタム CA                    | —                |
| `tls_insecure`             | TLS 証明書検証をスキップ（開発用のみ）                  | `false`          |

### サーバー (`config/server.yaml`)

| キー              | 説明                                              | デフォルト     |
| ----------------- | ------------------------------------------------- | -------------- |
| `listen_addr`     | サーバーの待ち受けアドレス                        | `0.0.0.0:9443` |
| `allowed_hosts`   | 接続を許可するホストのリスト。**空 = 全宛先許可** | `[]`           |
| `dial_timeout_ms` | 宛先への接続タイムアウト（ms）                    | `10000`        |
| `tls_cert_path`   | TLS 証明書パス（空 = ws://）                      | —              |
| `tls_key_path`    | TLS 秘密鍵パス（空 = ws://）                      | —              |
